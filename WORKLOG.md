## 2026-07-06 追記: アカウント接続・半透明・タブ整理

### 今回やったこと

- **Codex アカウント接続**: `~/.codex/sessions/**.jsonl` 末尾の `token_count` イベントにある
  `rate_limits`（primary=セッション5h窓 / secondary=週間窓、used_percent と resets_at、plan_type）を
  読み取り、mac 版と同じ「セッション◯%残り / ◯後にリセット」「週間」「プラン (Plus等)」を表示するようにした
  （`Services/CodexRateLimitReader.cs`）。ネットワークアクセスなし・ローカルログのみ。
- **Claude アカウント接続**: `~/.claude/.credentials.json` の OAuth トークンで
  `GET https://api.anthropic.com/api/oauth/usage`（`anthropic-beta: oauth-2025-04-20`）を呼び、
  five_hour / seven_day / seven_day_opus・sonnet / extra_usage / subscriptionType を表示
  （`Services/ClaudeOAuthUsageClient.cs`）。トークンは Anthropic 公式 API 以外へ送らず、表示・ログにも出さない。
  実機で Team プラン・セッション/週間の実数値取得を確認。
- **不要タブ削除**: Cursor / Gemini / Copilot のダミータブを廃止。タブは有効なプロバイダーだけを
  config (`providers[].enabled`) から動的生成。OpenAI タブは Admin API key がある場合のみ表示。
  既定 config の openai は enabled:false に変更。
- **半透明**: WS_EX_LAYERED + UpdateLayeredWindow の per-pixel アルファ描画に全面移行し、
  背後に SetWindowCompositionAttribute の ACCENT_ENABLE_ACRYLICBLURBEHIND（ラベンダーティント）、
  角丸は DWMWA_WINDOW_CORNER_PREFERENCE=ROUND（Win11）。アクリル不可環境では自動でほぼ不透明にフォールバック。
  `CODEXBAR_NO_ACRYLIC=1` で無効化、`CODEXBAR_DEBUG=1` で `%TEMP%\codexbar-debug.log` に診断出力。
- **文字被り修正**: 固定 468x840 の Form + 子 Control 構成をやめ、論理座標 (幅384) で組んで
  DeviceDpi/96 でスケールする方式に変更（PerMonitorV2 明示）。高さはコンテンツから動的算出。
  行高も全体的に広げ、タブ・使用率・メッセージ行の下端切れを解消。
- 使用状況ダッシュボード / ステータスページの行を実リンク化（プロバイダー別 URL をブラウザで開く）。

### 検証

- 実機起動し、PrintWindow / BitBlt(CAPTUREBLT) キャプチャで以下を目視確認:
  - タブが Codex / Claude の2つだけになり、選択タブ・ホバーが機能。
  - Codex: Plus プラン、セッション 0% 残り (58m 後にリセット)、週間 75% 残り、過去30日トークン。
  - Claude: Team プラン、セッション 40% 残り (4h33m 後にリセット)、週間 71% 残り、追加使用量 $0.00/$0.00。
  - 角丸 + アクリルの半透明パネルで文字の重なりなし。
